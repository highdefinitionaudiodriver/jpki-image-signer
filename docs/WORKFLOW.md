# クリエイター向け運用ガイド — 作品の真贋証明ワークフロー

JPKI Image Signer を **イラスト・写真・デザイン作品の真贋証明**に使うための
具体的な運用シナリオ集です。

> 💡 「自分の作品です」「改ざんされていません」を **国の認証局（公的個人認証サービス）の署名鍵** で示すための実用ガイドです。
> ブロックチェーン NFT のように身元担保が不明な仕組みではなく、**国が発行した署名鍵による真正性証明**で、技術的にも法的にも筋が通ります。

---

## 🎯 こんな悩みに応えます

| 悩み | このツールの答え |
|---|---|
| AI 生成物との見分けがつかなくなった | 自分のカードで署名 → 検証時に「公的認証された人物」と表示 |
| 盗用されて再投稿された | 受け取った相手が `.jpkiimg` の検証で発行日時・元データを確認可能 |
| 同人誌・物販で「本物の作者」と証明したい | 物販イベント時に署名付き画像を販売、購入者は無料で検証可能 |
| 委任・契約書を画像化して送る必要がある | 個人間で「誰が承諾したか」を残せる |

## 📋 シナリオ 1: SNS 投稿用の作品に署名する

### 手順

1. 完成した作品 (`my_artwork.png`) を準備
2. JPKI Image Signer を起動 → 画像を GUI にドラッグ&ドロップ
3. マイナンバーカードをリーダーにセット → PIN 入力
4. `my_artwork.png.jpkiimg` が生成される

### 投稿時のワークフロー

```
公開用にエクスポート: my_artwork.png         （SNS 投稿用）
真贋証明の元: my_artwork.png.jpkiimg          （別保管、必要時に配布）
```

**投稿テンプレート例**：

> 新作です。本人作の証明（マイナンバーカード署名）が必要な方は DM ください。検証専用ページ: <Github Pages の検証ツール URL>

### 配布する `.jpkiimg` の管理

- **クラウドストレージに置く**（Google Drive、Dropbox 等の限定共有 URL）
- 公開リポジトリ（GitHub 等）への push は **厳禁** — 内包する電子証明書に氏名・住所が含まれます
- SNS への直接アップロードも **厳禁** — DM や限定公開 URL 経由のみ

## 📋 シナリオ 2: 同人誌・物販イベントでの真贋証明

### 事前準備

1. 販売予定の表紙画像・サンプル画像に署名
2. 検証用ページの URL を準備（GitHub Pages にホスティング推奨）
3. 名刺やフライヤーに **検証 URL の QR コード**を印刷

### イベント当日

- 購入者がスマホで QR コードを読み取り → 検証ページに
- `.jpkiimg` ファイルを共有（USB / AirDrop / 限定 URL 等）
- 購入者の端末で **完全ローカル検証**（ネットワーク通信なし、サーバー不要）
- ✅ 緑カードで「○○（作家本名）」と表示 → 真贋確認完了

### 物販後の継続効果

- 数年後の再販時にも同じ `.jpkiimg` で「同じ作者の作品」を証明可能
- 二次創作トラブル時の「自分が先に公開した」証拠としても機能

## 📋 シナリオ 3: AI 生成と区別するための「人間作家」証明

### 投稿ワークフロー

| ステップ | 内容 |
|---|---|
| 1. 制作 | 自分でイラスト・写真を撮影／作成 |
| 2. 署名 | 完成画像に JPKI Image Signer で署名 → `.jpkiimg` |
| 3. 公開 | 公開用画像 (.png/.jpg) を SNS 投稿 |
| 4. 検証窓口 | プロフィールに「本人作証明 検証 URL」を記載 |
| 5. 要求対応 | 検証希望者には DM 等で `.jpkiimg` を限定送付 |

### バッジ運用の提案

プロフィール画像の隅に **「JPKI 署名済」バッジ**を入れる運用が増えています：

```
■ プロフィール冒頭の例
○○ / イラストレーター
🪪 JPKI 署名対応（本人作証明 DM 可）
検証ツール: <URL>
```

## 📋 シナリオ 4: コミッション・依頼受注時の作者証明

依頼者から「本当にあなたが描いたんですよね？」と確認されたとき：

1. 完成画像と一緒に `.jpkiimg` を納品
2. 依頼者は検証ツール（[jpki-web/verifier](https://github.com/highdefinitionaudiodriver/jpki-web/tree/main/verifier)）で確認
3. ✅ 緑カード = 公的認証された人物による作品 と確認できる

法人クライアント・出版社向けには特に有効です。

## 🔒 セキュリティと注意事項

### 絶対にやってはいけないこと

- ❌ `.jpkiimg` を **GitHub・SNS で公開**する（内包証明書に氏名・住所等が含まれる）
- ❌ 検証スクショで `signer.subject_cn` を全部見せる（識別符号 17 桁が露出）
- ❌ PIN を画面共有時に入力する
- ❌ 公開サーバー（VPS 等）にこのツールをデプロイする（PIN がネットワークを越える）

### 推奨運用

- ✅ `.jpkiimg` は **限定共有 URL** または **DM** でのみ配布
- ✅ 検証側ツールは [jpki-web/verifier](https://github.com/highdefinitionaudiodriver/jpki-web/tree/main/verifier)（静的 HTML）を **自分のドメインに置く**
- ✅ 投稿時に「本人作証明 検証可能」と書く（バッジ化）
- ✅ 検証希望者には **公開画像と `.jpkiimg` のペア**を送る

## 💰 検証側は無料・署名側は本人カードが必要

- **検証だけしたい人**：[ブラウザ用検証ツール](https://github.com/highdefinitionaudiodriver/jpki-web/tree/main/verifier) を開いて `.jpkiimg` をドラッグ&ドロップ。**カード不要・インストール不要・無料**
- **署名したい本人**：マイナンバーカード（署名用電子証明書が有効なもの）＋ PC/SC 対応リーダーが必要
